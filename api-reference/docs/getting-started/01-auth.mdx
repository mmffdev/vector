# Authentication

Vector uses JWT bearer tokens issued by `POST /api/auth/login`.

## Login

```http
POST /api/auth/login
Content-Type: application/json

{
  "email": "user@example.com",
  "password": "..."
}
```

**Response** — `200 OK`

```json
{
  "access_token": "<jwt>",
  "user": {
    "id": "...",
    "email": "user@example.com",
    "role": { "code": "padmin", "label": "Project Admin" },
    "permissions": ["portfolios.create", "portfolios.list", ...],
    "is_active": true
  }
}
```

**Errors** — See [Error Handling](./05-error-handling.mdx):
- `401` Invalid credentials
- `403` Account inactive or locked
- `423` Account locked (too many failed attempts)

## Using the token

Pass the token as a Bearer header on every authenticated request:

```http
Authorization: Bearer <jwt>
```

## Token lifetime

Tokens expire after **24 hours**. Re-authenticate to get a new token.

## SSE endpoints

SSE endpoints (`text/event-stream`) do not support `EventSource` because that API cannot set custom headers. Use `fetch()` with a manual stream reader instead:

```ts
const res = await fetch(url, {
  headers: { Authorization: `Bearer ${token}` },
});
const reader = res.body!.getReader();
```
