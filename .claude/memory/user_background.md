Rick studied UX and Art History at university. His professional background is predominantly as an Agile Coach and transformation lead for 20+ years, with earlier experience in small-scale web design. He has no formal training as an architect, engineer, or computer scientist.

This background explains several patterns in how he works:
- Thinks in systems and process before implementation — built the Planka story system, 7-gate acceptance criteria, boot files, and memory system before focusing on product features
- UX training shows in semantic token naming choices (`--ink`, `--canvas` rather than `--gray-100`), accessibility instincts, and user-outcome framing
- Art History background informs aesthetic sensibility — palette relationships, colour polarity, vibe taxonomy in the theme system
- Transformation experience means strong discipline around defining done, decomposing correctly, and not shipping partial features

**How to apply:** Frame technical explanations in terms of systems, outcomes, and user experience — not code mechanics. His instincts on architecture and UX are sound; defer to them. The implementation gap is filled by Claude.
