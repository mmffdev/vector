# SSH ‚Äî reference

> Last verified: 2026-04-21

Host aliases, keys, and tunnel contracts for reaching `mmffdev.com`.

## Key

- `~/.ssh/id_ed25519` ‚Äî the key authorised on `root@mmffdev.com`. All `mmffdev-*` aliases use it implicitly via `IdentitiesOnly yes` or plain default-key lookup.
- Generated by `dev/scripts/ssh_manager.sh` if absent.
- Public key is installed on the server via `ssh-copy-id` (handled by the setup script).

## Host aliases

Three live blocks in `~/.ssh/config`:

### `mmffdev.com` ‚Äî Plesk-managed user

```
Host mmffdev.com
    HostName mmffdev.com
    User mmffdev.com_cnzy9bltlti
    IdentityFile ~/.ssh/mmffdev
    IdentitiesOnly yes
```

This is the Plesk-provisioned user for file uploads to the webspace. **Not** for DB or system admin ‚Äî use `mmffdev-admin` for those.

### `mmffdev-pg` ‚Äî multi-service tunnel

```
Host mmffdev-pg
  HostName mmffdev.com
  User root
  LocalForward 5434 localhost:5432   # Postgres (mmff_vector)
  LocalForward 8081 localhost:8081   # Adminer
  LocalForward 15672 localhost:15672 # RabbitMQ management
  LocalForward 9000 localhost:9000   # Portainer
  LocalForward 3333 localhost:3333   # Planka kanban board
  ServerAliveInterval 30
  ServerAliveCountMax 3
  ExitOnForwardFailure yes
```

Use: `ssh -N -f mmffdev-pg` to open all tunnels. `ExitOnForwardFailure yes` means if any port is already taken locally, `ssh` exits with an error instead of silently connecting without the forward.

### `mmffdev-admin` ‚Äî interactive / one-off root shell

```
Host mmffdev-admin
  HostName mmffdev.com
  User root
  ServerAliveInterval 30
  ServerAliveCountMax 3
```

Use: `ssh mmffdev-admin` for interactive sessions, or `ssh mmffdev-admin '<cmd>'` for one-offs.

## Setup script

[`dev/scripts/ssh_manager.sh`](../dev/scripts/ssh_manager.sh) creates the key, pushes it to the server, appends the two `mmffdev-*` blocks (if missing), and starts the tunnel. Safe to re-run ‚Äî every step confirms before acting.

## libpq gotcha (reminder)

macOS installs libpq as keg-only. `psql` and `pg_dump` are at `/opt/homebrew/opt/libpq/bin/`, NOT on PATH. The setup script appends a PATH export to `~/.zshrc` but scripts should use absolute paths regardless.
