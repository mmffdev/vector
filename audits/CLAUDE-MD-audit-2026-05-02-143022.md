Then flatten MEMORY.md to just reference these files by category.

## Optimization Opportunities

### 🎯 Opportunity #1: Add shell functions to bin/planka
**Current**: `./.claude/bin/planka` is a Python script
**Recommendation**: Create `./.claude/bin/hooks` or `./.claude/bin/functions.sh` with zsh functions for common operations:
- `mcp-auth` — quick MCP reauthentication
- `tunnel-check` — verify DB tunnel status
- `env-check` — verify ACTIVE_BACKEND_ENV marker
This avoids repeated setup commands in terminal sessions.

### 🎯 Opportunity #2: Consolidate research command chain
**Current**: 3 separate command files handle research:
- `c_research.md` — crawl agent
- `c_addpaper.md` — shorthand
- `c_write-research-paper.md` — CREATE step
**Recommendation**: Keep these as separate leaves, but add a parent `c_c_research_*.md` index that maps the workflow:
1. Use `<addpaper>` for quick web-only research (→ RNNN.json)
2. Use `/research` for deep crawl + compile (→ comprehensive report)
3. Use `/writepaper` to synthesize and create Planka cards

### 🎯 Opportunity #3: Add librarian audit schedule
**Current**: `<librarian>` runs on-demand; librarian-log.jsonl records runs
**Recommendation**: Document a suggested cadence in c_librarian.md:
- **After `/stories` cards ship**: run `<librarian> full` to sync docs
- **Weekly**: run `<librarian>` with no args (scans last 7 days by default)
- **Before launches**: run `<librarian> security` to flag new vulns
This keeps the golden source fresh without manual checkpoints.

### 🎯 Opportunity #4: Add a c_shortcuts_reference.md leaf
**Current**: `<?>` command regenerates `dev/shortcuts.html` from command + skill docs
**Recommendation**: Create a `docs/c_c_shortcuts_reference.md` that documents:
- How to register a new command shortcut (template)
- How to register a new skill hotkey
- Existing shortcut map (machine-readable table)
- Conflict resolution rules
This prevents accidental hotkey collisions and makes onboarding clearer.
