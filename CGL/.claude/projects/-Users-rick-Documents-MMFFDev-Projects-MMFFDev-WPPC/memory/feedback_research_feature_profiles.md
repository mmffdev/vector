Research skill must produce feature profiles when researching products/tools/platforms — what each feature does, how it works, what problem it solves, how the source categorises it. Pure intelligence gathering.

**Why:** The research page is a reference library. Mapping to our project scope is a separate deliberate decision, not something baked into the research output. User stories and gap analysis pollute the research with assumptions.

**How to apply:** When `/research-crawl` or `/uresearch` targets a product/tool, use Format A (Feature Profile) from the agent protocol. Only use Format B (General Analysis) for abstract topics/concepts/questions. Never generate backlog items, user stories, or implementation recommendations as part of research output.
