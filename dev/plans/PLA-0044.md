## Why this plan exists

Topology data is currently walked **four times independently** across the codebase (and a fifth surface — user-to-node assignment — needs to land soon), with subtle drift between each walk that produces user-visible bugs:

| Surface | File | Walk shape | Orphan policy | Sort | Quirks |
|---|---|---|---|---|---|
| Canvas card map (dagre) | `app/components/topology/layoutWithDagre.ts` | `walk(roots)` recursion + visibleEdges loop | drop (filter on `archived_at IS NULL` via input) | by `position` | inline, not extracted |
| Canvas tree state (table view) | `app/components/topology/useTopologyTreeState.ts` | builds `childrenOf` Map only | drop (filter on `archived_at IS NULL`) | by `position` | doesn't flatten — consumers re-walk |
| Topology flyout (canvas left rail) | `app/components/TopologyTreeFlyout.tsx` | `flatten()` recursion with `ancestorMoreChildren` path | none — relies on input being clean | by input order (caller-sorted) | depth-0 quirk: `pathMoreChildren = []` at root, special-cased |
| Scope rail (chrome sidebar) | `app/components/ScopeRail.tsx` | `buildTree()` recursion with `ancestorMoreChildren` path | **re-roots orphans** — produces phantom "D" rows | by `label` | bug: shows orphans the canvas correctly drops |
| **(future) User-to-node assignment picker** | new — `app/components/topology/UserNodeAssignment.tsx` | TBD — would have re-written the walk a fifth time | TBD | by `position` | gadmin picks which topology node(s) a user is assigned to / scoped against |

The drift is not stylistic — each version answers the same questions ("what's visible? at what depth? with what spine geometry?") with different code paths. R052 + R053 research validated that the *data model* is universal (single-FK ownership, recursive-CTE walks); the duplication is purely an artefact of incremental delivery.

This plan extracts a single `walkTopology()` engine that:

1. **Drives every existing consumer** (canvas, table, flyout, rail) with one walk, one orphan policy, one sort knob.
2. **Drives the BFF tree response** (`/_site/topology/tree`) so server-side answers to "what's visible to this user in this scope" use the same algorithm — no UI/server drift.
3. **Mirrors to Go** (`backend/internal/shared/topology/walker.go`) with golden-fixture parity tests so the eventual external `/samantha/v2/topology/tree` route is the *same algorithm*, not a re-implementation.
4. **Powers the user-to-node assignment picker** (new surface, FE-POR-0003.9.10) where a gadmin selects which topology node(s) a user is assigned to or scoped against. The picker is a tree-shaped multi-select; the walker provides the visible-rows + edges + collapse semantics — only the row renderer is bespoke (checkbox cell instead of a name).

The engine is the foundation; subsequent topology work (move semantics, ancestor breadcrumbs, materialised paths) extends it instead of branching again.

## Engine contract

**Single entrypoint.** `walkTopology` is the only public function. Consumers that need just the parent→children map destructure `result.childrenOf` from the result — no separate `buildChildrenOf` export. Edges are emitted during the same recursion that produces rows (one pass over the tree, not two).

Orphans (nodes whose `parent_id` resolves to a missing/filtered node) are dropped — matching the canvas's pre-existing behaviour. A consumer that wants a different orphan policy (e.g. re-root) supplies a `filter` that pre-resolves orphans; the walker stays single-policy.

Whether depth-0 rows draw a spine column is **renderer business**, not walker business. The walker emits `ancestorMoreChildren` for every row regardless of depth; renderers that want a flush-left root (rail) simply skip drawing the column at depth 0. This removes a fork point from the engine.

### Types (TS)

```ts
// Minimum surface any topology-like node must expose. Generic so the walker
// works for OrgNode (canvas data) and MyGrant (rail data) alike. Callers
// whose underlying record uses a different id key (e.g. MyGrant.node_id)
// adapt at the call site by mapping into this shape.
export interface TopologyNode {
  id: string;
  parent_id: string | null;
  archived_at?: string | null;
  position?: number;   // present on OrgNode; added to MyGrant in FE-POR-0003.9.8
}

export interface FlattenedRow<T> {
  node: T;
  depth: number;
  isFirst: boolean;
  isLast: boolean;
  hasChildren: boolean;
  collapsed: boolean;
  // Length = depth. Entry for the immediate parent is omitted because the
  // row's own elbow/T paints that column. Used by SVG spine renderers
  // (rail + flyout) to paint pass-through verticals for ancestors that
  // still have visible siblings below this row's subtree.
  ancestorMoreChildren: boolean[];
}

export interface WalkOpts<T extends TopologyNode> {
  collapsed: Set<string>;              // node IDs the user has hidden
  sort: (a: T, b: T) => number;        // canvas: byPosition; rail: byLabel until .9.8 lands position
  filter?: (n: T) => boolean;          // default: archived_at IS NULL
  maxDepth?: number;                   // default 12 (matches DescendantNodeIDs); beyond cap: cycle-truncated row
}

export interface WalkResult<T> {
  rows: FlattenedRow<T>[];
  visibleIds: Set<string>;
  visibleEdges: Array<{ source: string; target: string }>;
  childrenOf: Map<string | null, T[]>; // sorted, filtered; nil key = roots
}

export function walkTopology<T extends TopologyNode>(
  nodes: T[],
  opts: WalkOpts<T>,
): WalkResult<T>;
```

### Types (Go mirror)

The Go mirror uses a **generic struct with accessor funcs** rather than a method-interface, so callers don't need to wrap their existing structs (`OrgNode`, `Grant`) just to satisfy a behavioural interface. `ChildrenOf` keys on `string` with the empty string `""` reserved for roots — Go disallows `*string` map keys with stable JSON serialisation.

```go
type FlattenedRow[T any] struct {
    Node                 T      `json:"node"`
    Depth                int    `json:"depth"`
    IsFirst              bool   `json:"is_first"`
    IsLast               bool   `json:"is_last"`
    HasChildren          bool   `json:"has_children"`
    Collapsed            bool   `json:"collapsed"`
    AncestorMoreChildren []bool `json:"ancestor_more_children"`
}

type WalkOpts[T any] struct {
    // Accessor funcs replace a method-interface; callers pass their struct
    // type as T and supply the field accessors. Keeps caller types free of
    // engine-specific methods.
    ID         func(n T) string
    ParentID   func(n T) string        // empty string = root
    Archived   func(n T) bool          // true → filtered out by default Filter
    Position   func(n T) int           // 0 if unset; comparator decides usage
    Collapsed  map[string]struct{}
    Sort       func(a, b T) int        // -1/0/+1
    Filter     func(n T) bool          // default: !Archived(n)
    MaxDepth   int                     // default 12
}

type WalkResult[T any] struct {
    Rows         []FlattenedRow[T]                 `json:"rows"`
    VisibleIDs   map[string]struct{}               `json:"visible_ids"`
    VisibleEdges []struct{ Source, Target string } `json:"visible_edges"`
    // Roots keyed under "" so the map serialises with a stable string key.
    ChildrenOf   map[string][]T                    `json:"children_of"`
    // Sorted root list, redundant with ChildrenOf[""] but cheaper for
    // callers that only need the top level.
    Roots        []T                               `json:"roots"`
}

func WalkTopology[T any](nodes []T, opts WalkOpts[T]) WalkResult[T]
```

**Fixture-as-contract**: golden fixtures in `dev/fixtures/shared/topology/*.json` are byte-identical inputs/outputs that both the TS Vitest suite and the Go test suite read. CI runs both; non-zero diff between TS output and Go output (after stable-key JSON serialisation) fails the build. This is how TS↔Go parity stays locked over time without codegen.
