# PLA-0045 — Shared Methods Catalogue + Soft-Reminder Hook

**Created:** 2026-05-12
**Status:** drafted
**Sibling:** PLA-0044 (first cataloguer)
**Phase:** PH-0005

---

## Scope

A persistent home, naming convention, and discoverability surface for **shared methods** — logic re-used across more than one call surface (Frontend React ↔ BFF Route Handler ↔ Public Go API).

Three deliverables:

1. **Directory contract** — `app/lib/shared/<domain>/` (TS, cross-runtime: browser bundle + Next.js Node route handler) and `backend/internal/shared/<domain>/` (Go) for any logic that must run in both Node and Go; `dev/fixtures/shared/<domain>/` for parity golden fixtures.
2. **`docs/c_shared_methods.md` catalogue** — single index of every shared method with: domain, TS path, Go path (if present), fixtures path, consumers, status (`stable` / `evolving` / `experimental`).
3. **Soft-reminder hook** — `.claude/hooks/shared-methods-reminder.sh` that fires on `Write`/`Edit` against `app/api/**/route.ts` or `backend/internal/**/handler.go` and emits a one-line nudge ("New handler under <path> — does this logic belong in `app/lib/shared/<domain>/` and `backend/internal/shared/<domain>/`? See `docs/c_shared_methods.md`.").

Out of scope: retrofitting historical shared methods (separate audit). This plan only establishes the substrate + first entry (PLA-0044's topology walker).

---

## Value

**Why this matters now.** PLA-0044 is the first time the project deliberately writes one method that runs in three places (React browser, Next.js Node BFF, Go public API). Without a catalogue, the second occurrence drifts: someone writes a parallel utility, fixtures get skipped, the lint trio from PLA-0039 catches the symptom not the cause. The catalogue exists so the *third* occurrence finds the existing pattern and the fourth one extends it.

**What changes.** Future PRs that add a new `route.ts` or new Go handler get a chat-time nudge to consider shared placement before duplicating. Reviewers have one file to scan to know whether a given primitive already exists. Memory has one rule pointing at one doc.

## Implementation plan

1. **Create directories** — `app/lib/shared/.gitkeep`, `backend/internal/shared/.gitkeep`, `dev/fixtures/shared/.gitkeep` (so the path exists before PLA-0044 lands).
2. **Write `docs/c_shared_methods.md`** with the table format below + linkage to `docs/c_c_transport_segregation.md` (PLA-0039) and `app/lib/shared/topology/walker.ts` (PLA-0044) as the first row.
3. **Add CLAUDE.md pointer** — one line under "Working practices" pointing at `docs/c_shared_methods.md`.
4. **Write hook** — `.claude/hooks/shared-methods-reminder.sh`; matcher on Edit/Write where `file_path` matches `app/api/.*/route\.ts$` or `backend/internal/.*/handler\.go$` AND no prior session note in the same turn. Hook reads `docs/c_shared_methods.md` and emits to `additionalContext` a one-line reminder.
5. **Wire hook** in `.claude/settings.json` `hooks.PostToolUse` array.
6. **Memory rule** — write `.claude/memory/feedback_shared_methods_home.md`; add MEMORY.md index line.
7. **Lint allow-list seed** — `dev/registries/shared_methods.json` so `lint:writer-boundary` (PLA-0007) and `lint:transport-segregation` (PLA-0039) recognise `app/lib/shared/**` as cross-import-safe.

## Acceptance criteria

| # | Criterion | Proven by |
|---|---|---|
| 1 | The three shared directories exist on disk with `.gitkeep`. | `ls app/lib/shared/.gitkeep backend/internal/shared/.gitkeep dev/fixtures/shared/.gitkeep` returns all three paths. |
| 2 | `docs/c_shared_methods.md` exists and lists topology walker as row 1. | `grep -l "topology walker" docs/c_shared_methods.md` finds the file. |
| 3 | CLAUDE.md links to the catalogue. | `grep "c_shared_methods.md" .claude/CLAUDE.md` returns a line under "Working practices". |
| 4 | Editing a new `app/api/foo/route.ts` triggers the soft-reminder hook. | Manual test: create a throwaway `app/api/_test_shared/route.ts`; hook fires in `additionalContext` containing "shared". Delete the test file after. |
| 5 | Editing an unrelated file (e.g. `app/components/X.tsx`) does NOT trigger the hook. | Same test against a component file: no hook output. |
| 6 | Memory rule is loaded at session start. | `.claude/memory/MEMORY.md` contains line for `feedback_shared_methods_home.md`. |
| 7 | Lint allow-list includes `app/lib/shared/**`. | `grep "app/lib/shared" dev/registries/shared_methods.json` returns the entry. |

---

## Risks

| Impact | Risk | Mitigation |
|---|---|---|
| 2 | Hook becomes noisy (fires on every route edit, not just new ones). | Matcher uses `Write` (new files) primarily; `Edit` only fires if the changed file is < 100 lines (proxy for new-ish). Hook output is one line — easy to ignore. |
| 1 | Catalogue drifts out of date. | Memory rule + librarian sweep on `docs/c_shared_methods.md` mtime monthly. |
| 2 | Lint allow-list creates cross-import escape hatch abused by callers. | `app/lib/shared/**` files are explicitly cross-runtime (no React, no Node-only API); CI lint enforces. |
| 1 | Hook bash script breaks on macOS vs Linux regex. | Use POSIX `grep -E` only, no `sed -E`, no `[[ =~ ]]`. |
