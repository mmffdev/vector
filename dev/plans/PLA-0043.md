# PLA-0043 — Topology scope clamp on artefact reads

**Feature area:** FE-POR-0002 (scope / role-grant chrome surface)
**Status:** Drafted — ready for `<stories>` decomposition.
**Research:** [R052 — Rally Dev scope model](dev/research/R052.json) (validates URL-first `?scope=<id>` direction; adopts Rally's scope-down convention).
**Builds on:** PLA-0042 (picker UI + `ScopeContext`) — that ships the URL/localStorage substrate; this plan makes the substrate **mean** something.

---

## Why this plan exists

PLA-0042 shipped the chrome picker — users can switch between topology nodes they hold a grant on, and the choice persists to `?scope=<node_id>` + localStorage. But picking a scope **does nothing yet**: the backlog, portfolio, dashboards, and every other read are still workspace-clamped only. This plan wires the active scope into those reads so a user holding a grant on the "Engineering" node sees only the artefacts attached to Engineering and its descendants.

R052 confirmed our direction is right: Rally's own WSAPI takes explicit `?project=…&projectScopeDown=true` query params on every read. We're making URL-first what Rally makes session-first.

---

## Scope of THIS plan

**In:**

1. **Schema** — add `topology_node_id UUID NULL REFERENCES topology_nodes(id)` to `vector_artefacts.artefacts`. Nullable so existing rows survive; FK keeps integrity.
2. **Descendant helper** — `orgdesign.Service.DescendantNodeIDs(ctx, rootNodeID) ([]uuid.UUID, error)`. Recursive CTE on `topology_nodes`; filters `archived_at IS NULL`; returns root + every live descendant. Sole-writer boundary unchanged (this is a read).
3. **Scope-down convention (hard)** — picking node X means "X and every descendant." No ancestor inclusion, no toggles. Adopted from Rally's `projectScopeDown=true` default. If a future "roll-up to parent" view is wanted, that's a separate plan.
4. **Artefact read clamp** — `artefactitemsv2.Service.List` accepts an optional `scope_node_id`; when present, adds `AND topology_node_id = ANY($descendants)` to the WHERE clause. Workspace clamp **stacks** (both required).
5. **BFF wiring** — `apiSite` automatically forwards the active `scope` from URL on every `/_site` read. No per-endpoint frontend change needed.
6. **Public-API wiring** — `/samantha/v2/work-items` documents the optional `?scope=<id>` query param in `openapi-v2.yaml`. Same shared service core (PLA-0039 transport segregation).
7. **Authorisation** — the scope clamp does not bypass auth. A user passing a `scope` for a node they don't hold a grant on gets `403 ROLE_SCOPE_FORBIDDEN`. Gadmin bypasses the grant check (matches PLA-0042 synthetic-grants path).
8. **Audit** — successful reads emit no audit (reads are not logged today); failed scope-permission checks emit a `scope_denied` audit row tying user → attempted node → endpoint.

**Deferred (NOT in this PR):**

- Backfilling `topology_node_id` on existing artefacts. Today's artefacts have no node assignment; new artefacts get a node via the artefact-create flow (see "Backfill strategy" below). Backfill happens organically as users open artefacts and assign them; a one-shot ETL is an explicit follow-up.
- **Write-side scope clamps** — POST/PATCH on artefacts. This plan is read-only. Writes get a separate plan because they need a "default scope" rule (does creating an artefact while scope=Engineering auto-assign `topology_node_id=Engineering`?).
- **Portfolio-item reads.** Same pattern, but a different read service (`portfolioitems`) — separate story under this plan, but listed as iteration-2.5 if the work-item path proves the pattern.
- **Scope filter for sprints, releases, timeboxes.** Same clamp shape, but `TimeboxManager`-backed; depends on the artefact path landing first because timeboxes reference artefacts.
- **Ancestor / sibling rollup views.** "Show all sibling teams' work" is a legitimate future ask but not a default. Out of scope until a real user surfaces it.

## Wire format

```jsonc
// GET /api/work-items?scope=<topology_node_id>&workspace=<workspace_id>
// Stacks: workspace clamp AND scope clamp both apply.
// Scope clamp = topology_node_id IN (scope + all descendants).
[
  {
    "id": "…",
    "title": "Refactor login flow",
    "workspace_id": "…",
    "topology_node_id": "…",   // NEW — may be null for un-scoped artefacts
    "…": "…"
  }
]
```

**Error shape (new):**

```jsonc
// 403 — actor has no grant on the requested scope
{
  "error": "scope_denied",
  "message": "You do not have access to this scope.",
  "scope_node_id": "…"
}
```

## Backfill strategy (deferred but stated here)

Existing artefacts have `topology_node_id = NULL`. The read clamp behaviour:

- **No `?scope=` on request** → return all artefacts matching workspace clamp (today's behaviour preserved).
- **`?scope=<id>` on request** → return only artefacts with `topology_node_id IN (descendants)`. Un-assigned artefacts are **excluded** when a scope is active.

This is deliberate: it preserves "All Items" mode (no scope) without forcing a backfill, while making the scoped view meaningful immediately. As users assign nodes to artefacts (via the artefact-edit form — a follow-up plan adds that field), the scoped views fill out organically.

A future plan can ETL un-assigned artefacts to a default node (e.g. workspace root) once we know what tenants actually want.

## Edge cases handled

| Case | Behaviour |
|---|---|
| No `?scope=` on URL | Today's behaviour. Workspace clamp only. |
| `?scope=` references a node the user has no grant on | 403 `scope_denied`. Audit row written. |
| `?scope=` references an archived node | 403 (treated as no grant — archived nodes are limbo per PLA-0006). |
| Gadmin user with `?scope=<any-node-in-subscription>` | Permitted. Matches PLA-0042 synthetic-grants path. |
| User holds grant on parent, requests scope on child | Permitted. Grants inherit **down** the tree on read. Rally-style. |
| User holds grant on child, requests scope on parent | **Denied.** Grants do NOT inherit up. (Avoids data leakage from child team → parent program.) |
| Scope query param malformed (not a UUID) | 400 `RequestInvalidID`. |
| Empty descendant set (leaf node with no children) | Clamp is `topology_node_id = <id>` — exact match only. |
| Artefact with `topology_node_id = NULL` while scope is active | Excluded from scoped view. Visible in unscoped view. |

## Authorisation model (the meat)

This is where the plan earns its keep. Three permission cases:

1. **Read with no scope** — actor must hold any grant in their subscription (today's check). Empty grants = empty workspace.
2. **Read with scope=X** — actor must hold a grant on **X or any ancestor of X**. Inherits down. Encoded in `permissions.CanReadScope`.
3. **Gadmin** — bypasses (2). Matches PLA-0042's synthetic-grants behaviour: gadmin sees the full subscription tree.

The "ancestor grant covers descendant scope" rule is the load-bearing decision. Rationale: if you're granted on "Engineering," you should be able to scope down to "Engineering > Backend > Auth" without needing a second explicit grant on Auth. Otherwise the grant system is unusable for any tenant deeper than two levels.

This is also why we explicitly **do not** allow the reverse (child grant → parent scope): a backend-team lead with a grant on "Engineering > Backend" can scope down into their own subtree, but can't scope up to "Engineering" and see frontend team's work.

---

## Why this isn't a header or session token

Rally uses session-stored project OID. We considered it; we're not doing it:

- **URL is bookmarkable.** `?scope=<id>` round-trips on link share, refresh, browser-back. Session state breaks all of that.
- **URL is debuggable.** Every API call carries it. No "why am I seeing different data than my colleague" mystery.
- **No write amplification.** Session-stored scope means an extra DB write on every picker change. URL is free.
- **Matches our existing convention.** `?workspace=`, `?archived=`, `?q=` all live in the URL today.

R052 says Rally chose session because their UI is a Java Servlet app from 2008. We have no such constraint.

## Verification

1. **Schema** — `psql -c "\d vector_artefacts.artefacts"` shows `topology_node_id UUID REFERENCES topology_nodes(id)`.
2. **Descendants helper** — Go test: tree of 5 nodes; root call returns 5 IDs; leaf call returns 1.
3. **Permission helper** — Go test matrix: grant-on-self/ancestor/none/gadmin × scope-on-self/descendant/ancestor.
4. **Read clamp** — fixture: 3 artefacts on node A, 2 on node B (sibling of A), 1 un-assigned. Request without scope → 6 items. Request `?scope=A` → 3. Request `?scope=B` → 2.
5. **Authorisation** — log in as a user with grant on node A only. `?scope=A` → 200. `?scope=B` (sibling) → 403. `?scope=A_child` (descendant) → 200.
6. **Gadmin** — log in as gadmin. `?scope=<any node>` → 200.
7. **Frontend round-trip** — switch picker to "Engineering"; URL becomes `?scope=<eng-id>`; backlog table refetches; visible rows are Engineering's subtree only. Switch to "All scopes" (clear) → all artefacts visible (matching today's behaviour).
8. **Audit** — failed permission check writes one row to `audit_log` with `event_code = scope_read_denied`.
